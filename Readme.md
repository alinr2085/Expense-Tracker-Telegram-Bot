# Exackerone Bot 💸

ربات تلگرامی مدیریت هزینه‌های شخصی، ساخته‌شده با Spring Boot. کل تعامل با ربات از طریق دکمه‌های شیشه‌ای (Inline Keyboard) و کیبورد اصلی تلگرام انجام می‌شه — بدون نیاز به تایپ دستورات پیچیده.

> ربات به‌طور کامل به **زبان فارسی** و با لحنی صمیمی و غیررسمی (محاوره‌ای) با کاربر صحبت می‌کنه.

---

## ✨ امکانات

- **ثبت تراکنش جدید** — مبلغ، نوع (هزینه/درآمد)، دسته‌بندی، تاریخ (با تقویم شیشه‌ای) و توضیحات، همه مرحله‌به‌مرحله و با دکمه
- **مشاهده‌ی تاریخچه** — نمایش ۱۰ تراکنش اخیر با جزئیات کامل
- **انتخاب و مدیریت تراکنش** — انتخاب یه تراکنش خاص از تاریخچه برای ویرایش یا حذف
- **ویرایش تراکنش** — تغییر مبلغ یا توضیحات یه تراکنش ثبت‌شده
- **حذف تراکنش**
- **خروجی اکسل (Excel)** — دریافت فایل کامل تاریخچه‌ی تراکنش‌ها به فرمت `.xlsx`
- **تقویم شیشه‌ای (Inline Calendar)** — انتخاب تاریخ تراکنش با ورق‌زدن ماه‌ها، بدون نیاز به تایپ دستی تاریخ
- **مدیریت مکالمه‌ی چندمرحله‌ای (Conversation State Machine)** — ربات یادش می‌مونه کاربر توی کدوم مرحله از فرآینده (مثلاً منتظر مبلغه یا منتظر توضیحاته)

## 🛠️ تکنولوژی‌ها و ابزارها

| ابزار / کتابخونه | کاربرد |
|---|---|
| **Java + Spring Boot** | فریم‌ورک اصلی برنامه (Dependency Injection، مدیریت کامپوننت‌ها) |
| **Spring Data JPA** | ارتباط با دیتابیس و کوئری‌های خودکار (Derived Query Methods) |
| **PostgreSQL** | دیتابیس اصلی برای ذخیره‌ی کاربران، تراکنش‌ها و دسته‌بندی‌ها |
| **Telegram Bots Java Library** (`org.telegram.telegrambots`) | ارتباط با Telegram Bot API (دریافت آپدیت‌ها، ارسال پیام و کیبورد) |
| **Long Polling** | روش دریافت پیام‌های تلگرام (بدون نیاز به وب‌هوک و دامنه‌ی عمومی) |
| **Lombok** | کاهش کدهای تکراری (Getter/Setter خودکار با `@Data`) |
| **Apache POI** | ساخت فایل خروجی اکسل (`.xlsx`) از تاریخچه‌ی تراکنش‌ها |
| **Maven / Gradle** | مدیریت وابستگی‌ها و build پروژه |

## 🏗️ معماری پروژه

پروژه از چند لایه‌ی مجزا تشکیل شده تا هر بخش مسئولیت مشخصی داشته باشه:

```
bot/
├── command/     → دستورات مستقل (مثل شروع مکالمه، لغو، دانلود فایل)
├── step/        → مراحل یک مکالمه‌ی چندقدمی (مثل گرفتن مبلغ، دسته‌بندی، تاریخ)
├── state/       → مدیریت وضعیت مکالمه‌ی هر کاربر (Session)
└── ...          → دیسپچر آپدیت‌ها، ساخت کیبوردها، ارسال پیام
```

### چطور یه پیام پردازش می‌شه؟
۱. تلگرام یه `Update` جدید می‌فرسته (پیام متنی یا کلیک روی دکمه‌ی شیشه‌ای)
۲. `UpdateFlow` تشخیص می‌ده این یه **دستور** (Command) هست، یا بخشی از یه **مکالمه‌ی در حال انجام** (Step)
۳. بر اساس `ConversationState` فعلی کاربر (که توی `SessionManager` نگه‌داری می‌شه)، درخواست به هندلر مناسب فرستاده می‌شه
۴. هر مرحله (`Step`) کار خودش رو انجام می‌ده و در صورت نیاز، کاربر رو به مرحله‌ی بعدی هدایت می‌کنه

این طراحی باعث می‌شه اضافه کردن یه دستور یا مرحله‌ی جدید، بدون دست‌زدن به منطق مرکزی، فقط با اضافه کردن یه کلاس جدید (که Spring خودش پیدا و ثبتش می‌کنه) ممکن باشه.

## ⚙️ راه‌اندازی محلی (Local Setup)

### پیش‌نیازها
- Java 17+
- PostgreSQL (یه دیتابیس خالی به اسم `expense_tracker` بسازید)
- یه توکن ربات تلگرام (از [@BotFather](https://t.me/BotFather))

### تنظیمات
پروژه از دو فایل تنظیمات استفاده می‌کنه:

**`application.properties`** (اطلاعات دیتابیس):
```properties
spring.application.name=Expense Tracker
server.port=8082

spring.datasource.url=jdbc:postgresql://localhost:5432/expense_tracker
spring.datasource.username=YOUR_DB_USERNAME
spring.datasource.password=YOUR_DB_PASSWORD

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
```

**`application.yml`** (توکن ربات، به‌صورت environment variable):
```yaml
telegram:
  bot:
    token: ${TELEGRAM_BOT_TOKEN}
```

قبل از اجرا، متغیر محیطی `TELEGRAM_BOT_TOKEN` رو ست کنید:
```bash
export TELEGRAM_BOT_TOKEN=your_actual_bot_token
```

### اجرا
```bash
./mvnw spring-boot:run
```
